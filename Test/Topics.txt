3.ครัวเรือนที่มีน้ำประปาคุณภาพดี -> IsHouseHoldGoodPlumbing 
ครัวเรือนที่เพิ่มมาจากกรณีตึกสูง ในข้อครัวเรือนทั้งหมด ให้พิจารณาคุณภาพน้ำประปาด้วย
    -> หาค่าเฉลี่ยร้อยละของ ครัวเรือนที่มีน้ำประปาคุณภาพดีต่อครัวเรือนทั้งหมด ในระดับตำบลที่ EA นั้นอยู่ (ถ้าไม่มีข้อมูลระดับตำบล ให้ใช้ระดับอำเภอแทน)
    -> ใช้ค่าเฉลี่ยร้อยละ ครัวเรือนที่มีน้ำประปาคุณภาพดีต่อครัวเรือนทั้งหมด ที่ได้มาหาครัวเรือนที่มีน้ำประปาคุณภาพดี
->query ea sampleType = "u" => sumIsHouseHoldGoodPlumbing * 100 /sumIshouseHold   => ได้ออกมาเป็น %
-> % * IsHouseHoldGoodPlumbing 
Note : SampleType="b" -> IsHouseHold != 0
-> SampleTye = "u" && IsHouseHold = 1


9.จำนวนบ่อน้ำบาดาล (สน.2) -> CountGroundWater
EA ที่มีบ่อบาดาลใน สน. 2 ซึ่งเป็นบ่อบาดาลสาธารณะ มากกว่า 100 บ่อ ถือว่าผิดปกติ
    -> ให้หาค่าเฉลี่ยของบ่อบาดาลของ สน.2 ในระดับตำบล (ตัด EA ข้อมูล EA ที่ผิดปกติออกก่อน) ของ EA ที่มีปัญหา
    -> ให้แทนค่า EA ที่ผิดปกติ ด้วยค่าเฉลี่ยบ่อบาดาลในระดับตำบล 
note: CountGroundWater ของ sampleType "c"
-> query sampleType "c" && CountGroundWater <= 100 เอามาหา เฉลี่ย CountGroundWater เอาไปใส่ให้ ea ที่ผิดปกติ

10.จำนวนประชากร -> CountPopulation -> (clear) เหลือตอนทำ sum
จำนวนประชากรในบาง EA มีมากเกินไป มากกว่า 20,000 คน ต่อ EA ถือว่าผิดปกติ (จะมีผิดปกติประมาณ 3 4 ตัวไปรันแก้ทีหลังได้)
    *countPopulation ใช้ Residential
    -> ให้นำค่าจำนวนประชากรจากแบบสอบถามด้านหน้าของโครงการน้ำมาใช้ เท่านั้น รายการที่ 45 (ไม่สนใจข้อมูลจากแบบสำรวจประชากรด้านหลัง) 
    และข้อมูลรายการที่ 45 ที่ดึงมาให้ยังพบข้อผิดปกติอยู่มาก เช่น มีจำนวนประชากรทั้งหมดเป็น 0 แต่มีจำนวนประชากรในวัยทำงาน เป็นต้น 
    จึงต้องทำการตรวจสอบส่วนนี้ให้ถูกต้องก่อนแก้ไขข้อมูล
        = countPopulation ใช้ Residential
    -> ให้แก้ไข EA ที่มีจำนวนประชากรที่มีมากกว่า 20,000 คน โดยแก้ไขรายครัวเรือน (ทีมไอทีแจ้งว่าเกิดจากการใส่ข้อมูลผิดมาของครัวเรือน) 
    ครัวเรือนที่แก้ไข ให้แทนค่าจำนวนประชากรด้วยค่าเฉลี่ยจำนวนประชากรของ EA นั้น (เป็นจำนวนเต็ม)
        = countPopulation หาค่าเฉลี่ยของประชากรใน EA ออกมา
            -ถ้าcountPopulation > countWorkingAge ให้ใช้ countPopulation
            -ถ้าcountPopulation < countWorkingAge ให้ใช้ countWorkingAge

11.จำนวนประชากรวัยทำงาน -> countWorkingAge
EA ที่มี CountPopulation > 0 แล้ว countWorkingAge ของ EA ต้องไม่เท่ากับ 0 คน
* จำนวนวัยทำงานที่น้อยผิดปกติ ไม่สนแล้วเอาแค่ countWorkingAge เป็น 0
    -> หาค่าเฉลี่ยร้อยละของ จำนวนประชากรวัยทำงานต่อจำนวนประชากรทั้งหมด ในระดับภูมิภาคที่ EA นั้นอยู่ (ตัด EA ที่มีปัญหาออก)
        = หา % countWorkingAge ต่อ countPopulation ในระดับภาค
    -> ใช้ค่าเฉลี่ยร้อยละ จำนวนประชากรวัยทำงานต่อจำนวนประชากรทั้งหมด ในระดับภูมิภาคที่ EA นั้นอยู่  ที่ได้มาหาจำนวนประชากรวัยทำงาน
        = เอา % ที่ได้มา * countPopulation ของ Ea นั้น

15.พื้นที่ชลประทาน -> FieldCommunity 
EA ที่มีพื้นที่ 300 ไร่ ถือว่าผิดปกติ เนื่องจากเป็นพื้นที่สาธารณะที่ชุมชนมารวมตัวกันเพื่อทำการเกษตรโดยใช้แหล่งน้ำเดียวกันเท่านั้น 
    -> หาค่าเฉลี่ยพื้นที่ชลประทานราย EA ระดับภูมิภาค 
        1. ภาคกลาง พื้นที่ชลประทานระดับ EA = 22 ไร่
        2. ภาคตะวันออกเฉียงเหนือ พื้นที่ชลประทานระดับ EA = 26 ไร่
        3. ภาคใต้ พื้นที่ชลประทานระดับ EA = 21 ไร่
        4. ภาคเหนือ พื้นที่ชลประทานระดับ EA = 25 ไร่
    -> เอาค่าเฉลี่ยของภูมิภาค ที่ EA นั้นๆอยู่ มาแทน 
์์Note: แปลงหน่วยด้วย
-> query FieldCommunity >= 300 groupReg

20.สถานที่ราชการที่มีน้ำประปาใช้ -> IsGovernmentUsage
EA ที่มีสถานที่ราชการ แต่ไม่มีสถานที่ราชการที่มีน้ำประปาใช้ หรือมีน้อยมาก ถือว่าผิดปกติ
    -> พิจารณาสถานที่ราชการทั้งหมด จากชื่อถนนที่สถานที่ราชการนั้นอยู่ 
    -> ถ้าถนนเส้นนั้นมีสถานที่ราชการที่มีน้ำประปาใช้ ให้ถือว่าสถานที่ราชการในถนนเส้นนั้นทั้งหมดเป็นสถานที่ราชการที่มีน้ำประปาใช้
์Note : ไม่มีชื่อถนน

21.สถานที่ราชการที่มีน้ำประปาที่มีคุณภาพมาตรฐาน -> IsGovernmentWaterQuality
พิจารณาตามคุณภาพน้ำประปาของสถานที่ราชการที่มีน้ำประปาใช้ในถนนเส้นเดียวกันนั้น
    -> ถ้าถนนเส้นนั้นมีสถานที่ราชการที่มีน้ำประปาใช้ และเป็นน้ำประปาที่มีคุณภาพมาตรฐาน 
    ให้ถือว่าสถานที่ราชการในถนนเส้นนั้นทั้งหมดเป็นสถานที่ราชการที่มีน้ำประปาที่มีคุณภาพมาตรฐาน
Note(20-21): ข้อมูลที่กรอกมือ มันไม่มีทางเหมือนกัน แล้วเราไม่ได้เก็บไว้ใน v5 ถ้าจะใช้จะต้องรันใหม่
-ถนน -> เค้าต้องใช้ชื่อถนนมาคำนวณ แล้วไม่ได้เก็บเป็น specific
